This Privacy Policy explains how SQL Lab processes personal data.

1. Data Controller

The operator of the deployed SQL Lab instance is the data controller.

2. Personal Data Collected

Username

Authentication credentials (hashed)

Container metadata and logs

User-generated SQL queries (processed transiently)

3. Purpose of Processing

Personal data is processed solely to:

Authenticate users

Provide access to isolated database environments

Manage containers and system operations

Ensure platform security

4. Legal Basis (GDPR)

Processing is based on:

Legitimate interest (Article 6(1)(f))

User consent where applicable (Article 6(1)(a))

5. Data Retention

User data is retained only while the account exists

Deleting a user removes associated containers and databases

6. Data Sharing

No personal data is shared with third parties.

7. User Rights

Users have the right to:

Access their data

Request correction or deletion

Withdraw consent where applicable

Requests should be directed to the platform operator.

8. Security Measures

Password hashing

Token-based authentication

Container isolation

Restricted administrative access

9. Changes

This policy may be updated. Continued use constitutes acceptance of changes.
